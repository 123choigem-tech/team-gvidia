# Before / After

## 작업 시각
- 원본 기준 시점: 2026-06-30 19:20 KST
- 자산 추가 시점: 2026-06-30 22:26 KST

## Before
- Home 화면은 기상 기사 CSV만 직접 확인해야 해서, 지표를 보려면 파일을 열어 수동으로 세어야 했다.
- 기사 수집 출처(`origin`)와 수집 시점(`crawled_at`)이 Home 화면에 바로 보이지 않아, 데이터가 어디서 왔는지 추적이 어려웠다.
- 파이프라인 실행 화면과 SST 분석 화면이 Home과 분리되어 있어 이동 동선이 길었다.
- SST 그래프가 `matplotlib` 기반이라, 다른 화면과 스타일이 조금씩 달라 보였다.
- 상단 예약 카드와 비상 CTA가 화면에 강하게 드러나지 않아, 첫 인상이 분산됐다.
- 제출용 Markdown 문서는 인코딩이 깨져 있어 그대로 읽기 어려웠다.
- 재사용 자산이 정리되지 않아, 다음 작업에서 그대로 가져다 쓰기 어려웠다.

## After
- Home에서 `EventStore`를 직접 조회해, 기사 수집 데이터의 기준 기간과 수집 시점을 바로 확인할 수 있게 했다.
- Home 업데이트 결과에 수집 출처(`origin`)별 개수도 보이도록 바꿔, 데이터 신뢰 흐름을 한눈에 보이게 했다.
- `pages/1_Home.py`에서 파이프라인 실행과 SST 분석 결과를 바로 연결해, 화면 이동 없이 흐름을 이어가게 했다.
- `pages/5_Pipeline.py`, `pages/5_SST_Timeseries.py`는 제거해 동선을 단순화했다.
- SST 그래프를 `plotly`로 바꿔, Home과 분석 화면의 시각 언어를 맞췄다.
- Home 상단의 `현재 분석 현황` 카드 영역을 정리해, 핵심 상태가 먼저 보이도록 했다.
- 버튼과 알림 UI를 체계적으로 정리해, 화면 간 톤이 맞게 보이도록 했다.
- 제출용 Markdown 문서를 UTF-8 기준으로 다시 작성해, 바로 열어도 읽히게 했다.
- `submit/assets/`에 재사용 가능한 산출물을 정리해, 다음 작업에서 바로 복붙하거나 재활용할 수 있게 했다.

## 효과
- Home 기준으로 기사 출처와 수집 시점을 바로 확인할 수 있어, 수동 검증 단계가 줄었다.
- 화면이 `Home + 분석/보고` 중심으로 정리되어 발표와 시연 흐름이 더 단순해졌다.
- SST 시각화 스타일이 통일되어, 화면 전환 시 이질감이 줄었다.
- 제출 문서와 자산이 정리되어, 이후 수정과 재사용 속도가 빨라졌다.

## 메모
- `submit/evidence/timestamps.txt`에 파일 수정 시각을 함께 남겨두면, 제출 시점과 작업 시점이 맞는지 바로 확인할 수 있다.
