# 프로젝트 재사용 메모

## 현재 구조에서 자주 재사용되는 축
- `src/`: 수집, 전처리, 분석, 보고서 생성의 핵심 로직
- `pages/`: Streamlit 화면
- `agents/`: 자동 보고서 및 보조 작업
- `scripts/`: 실행용 CLI 래퍼와 보조 스크립트
- `utils/`: 시각화, 채팅, 공통 유틸

## 재사용할 때 확인할 포인트
- 입력 경로가 `data/input`인지, 결과 경로가 `data/results`인지 확인
- CSV 인코딩은 `utf-8` 또는 `utf-8-sig`를 우선 사용
- 보고서나 화면에서 경로가 바뀌면 `pages/`와 `agents/`를 같이 점검
- 생성물은 제출 폴더가 아니라 프로젝트 루트 또는 `src/`에 두고, 제출본은 `submit/`에 별도 정리

## 팀 공유용 한 줄
- "반복 실행 명령과 보고서 프롬프트는 `submit/assets/`에 모아두면 다음 작업 때 바로 재사용할 수 있다."
