# 재사용 프롬프트 팩

## 1) 작업 로그용 프롬프트
다음 형식으로 실제 작업 내용을 기록해줘.
- 작성자, 목표, 핵심 지시문, 사용한 기법, 결과, 막힘과 해결
- 핵심 지시문은 사용자가 실제로 말한 문장을 그대로 인용
- 한 작업이 끝날 때마다 1개 항목만 추가

## 2) Before / After용 프롬프트
현재 작업의 Before와 After를 비교해서 정리해줘.
- Before: 기존 수작업 방식
- After: 에이전트화 후 방식
- 효과: 시간, 반복 횟수, 실수 감소, 일관성 개선 중 해당되는 것만 적기

## 3) 제출 점검용 프롬프트
`submit/CHECKLIST.md` 기준으로 빠진 항목을 찾아줘.
- 파일명
- 위치
- 누락 여부
- 보완 방법

## 4) 복붙용 요약 문구
> "반복 수작업을 자동화해서 Home에서 수집 출처, 분석 결과, 보고서를 한 흐름으로 보이게 정리했다."

> "재사용 자산과 제출 문서를 같이 정리해 다음 작업에서도 바로 가져다 쓸 수 있게 했다."
